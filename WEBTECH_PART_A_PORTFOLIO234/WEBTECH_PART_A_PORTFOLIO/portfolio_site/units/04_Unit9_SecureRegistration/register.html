Unit 9 Task 1 – Secure Registration & Validation

This form demonstrates stronger password rules and consent-based registration. Client-side validation improves usability, but security requires server-side validation and password hashing in production.

Registration Form

Policy: ≥12 chars; uppercase; lowercase; number; symbol.

Security rationale (brief)

  • Plain-text password storage is unsafe because it exposes users immediately if a database is compromised.
  • Production systems should store passwords using strong, salted hashing (e.g., bcrypt/Argon2) and validate on the server.
  • Client-side validation supports usability but does not prevent malicious requests.
